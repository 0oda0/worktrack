# Воркфлоу разработки

Работа идёт спринтами (список — в [backend-plan.md](./backend-plan.md)).
Каждый спринт проходит фиксированный цикл:

1. **Реализация** — расчётная/бизнес-логика через TDD: сначала тест, потом код.
2. **Тестирование** — `pytest` полностью зелёный + ручная проверка эндпоинтов
   (Swagger `/docs` или curl); для фронтенд-спринтов — проверка в браузере,
   включая мобильный вьюпорт 375px.
3. **Код-ревью** — ревью сделанного за спринт кода свежим взглядом:
   корректность, права доступа (worker/leader/admin), часовые пояса,
   краевые случаи. Правки по итогам ревью — в этом же спринте.
4. **Коммит** — один коммит на спринт.
5. **Стоп** — следующий спринт начинается только после команды «дальше».

## Правила коммитов

- Сообщение: краткая строка-итог + список изменений по делу.
- **Без упоминания ИИ-инструментов**: никаких Co-Authored-By, "Generated with",
  ссылок на Claude/нейросети и т. п.
- Пуш в `origin/rewrite/fastapi` после каждого коммита.

## Ветки

- `rewrite/fastapi` — основная ветка переписывания, история сохранена от `main`.
- `main` (старый код) не трогаем до финального мержа.
